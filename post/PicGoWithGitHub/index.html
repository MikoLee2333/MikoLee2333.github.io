PicGo配合GitHub搭建属于自己的图床

最近由于Hlae中文站的开通，相信有发帖需求的各位最近一定被markdown如何添加图片烦到了，这里我来给大家写一个简单地gGitHub图床的教程，希望可以帮助到大家。

下面正式开始：

第一步：GitHub账号配置

首先你得拥有一个GitHub账号，没有的话你得先申请一个，点此处申请GitHub账号

申请账号后，请登录账号，接下来我们正式开始GitHub图床的配置

  1. 首先来到账号首页

​ 先点击 头像旁边的小三角 ，接着点击 Your repositories ,进入 Repository 界面

​ 点击 New Repository 新建一个仓库

  1. 从上往下依次为：

Repository name -> 仓库名		//此项必填，后面会用到

Description -> 仓库描述			//可填可不填

Public or Private -> 仓库权限，公有或私有，具体自己去了解					//两个都可以

Initialize this repository with a README -> 为仓库初始化一个说明文档	//可选可不选

Create Repository -> 创建仓库 	//不多解释了

​ 按照上面的信息填完，我们需要 Repository name ，后面 PicGo的配置 会用到它

  1. 获取图床的Token

    点击头像旁边的小三角 ，然后点击 Settings ，进入个人设置页面

​ 点击 Developer settings ，进入开发者设置页面

​ 点击 Personal access tokens ，进入token管理页面

​ 点击 Generate new token ，创建新的token

​ note栏为token的说明，你可以填你想填的任何东西

​ 然后勾选 repo ，将页面拉到最下方，点击 Generate token ，创建token

​ 此时出现的这串代码就是token，这个只会出现一次，请将其保存下来，接下来 PicGo的配置 我们会用到它

PicGo的安装与配置

PicGo的安装

​ 下载-> PicGo <-

​ MacOS下载.dmg格式的文件，Windows下载.exe格式的文件

​ 安装一直next就行

PicGo的配置

  1. 安装结束后打开软件设置页面

  1. 勾选GitHub图床，当然你也可以使用其他图床，但是这里我只说GitHub图床哈

    ​ 接下来，点击 图床设置 -> GitHub图床

​ 从上往下五项分别是：

  • 仓库名，即前面的 owner / Repository name，例如我的，就是 MikoLee2333/PicGoPhoto

  • 分支名，默认为 master ，自行填写即可

  • Token，即上面我们让保存的一长串字符，复制填入即可

  • 存储路径，自行填写，可以参照我的来填，之后上传图片后会在你的GitHub仓库生成对应的文件夹，你上传的图片也会保存在里面

  • 自定义域名，此处请注意，格式有要求，格式必须为

    https://raw.githubusercontent.com/Owner/Repository name/master，

    例如我的则是https://raw.githubusercontent.com/MikoLee2333/PicGoPhoto/master

    该域名则是你将图片上传到GitHub图床后的链接

  1. 其他设置

    点击 上传区 ，可以看到，图片上传后返回的格式默认是markdown的图片链接格式，如此，就可以随心所欲的使用markdown进行图片的插入了

​ 点击 PicGo设置 ，进入PicGo设置页面，打开 开机自启 选项

​ 点击 修改快捷键 ，进入快捷键设置界面

​ 点击 编辑 ，修改快捷键，设置成你喜欢的快捷键

到这里，一个配合PicGo使用，能够迅速上传GitHub图库就大功告成了。

至于截图软件，就看你们自己的选择了，市面上的截图软件很多，你甚至可以用QQ截图，这里推荐的截图软件名字叫做 ShareX
