关于QT MSVC编译器使用中文出现“C2001：常量中有换行符”的情况

2019-10-23 #QT/C++
​ 最近因为公司的项目是使用的QT+C++，所以在跟着《QT5.9开发指南》开始自学QT，在2.3.2章节教程的中文编码遇到了问题，在下面代码的位置 chkBoxBold = new QCheckBox(tr(&quot;粗体&quot;)); chkBoxUnder = new QCheckBox(tr(&quot;下划线&quot;)); chkBoxItalic = new QCheckBox(tr(&quot;斜体&quot;)); ​ 编译报错 C2001：常量中有换行符 ，在网络上查询资料及部分博客后，总结了以下几种解决方法： ​ 1、修改QT设置。 ​ 具体位置：Tools-&gt;Options-&gt;Text Editor-&gt;Behavior-&gt;File Encodings-&gt;UTF-8 BOM，将其修改为Add If Encoding is UTF-8 。 ​ 然后点击Apply，再点击OK，最后重启QT。此时你就会发现QT中文编译将不会报错，但是中文却变成乱码了。 ​ 解决乱码方法有以下两种： ​ 1-1 将代码中每一个中文都用用QStringLiteral()括起来（麻烦，不建议）； ​ 1-2 在对应的.cpp文件前加上#pragma execution_characte上r_set(&quot;utf-8&quot;)。 ​ 2、将中文改为英文（笑） ​ 3、在代码中使用中文的地方将中文使用 QString::fromLocal8Bit()括起来，如下所示： btnOK = new QPushButton(QString::fromLocal8Bit(&quot;确定&quot;)); btnCancel = new QPushButton(QString::fromLocal8Bit(&quot;取消&quot;)); btnClose = new QPushButton(QString::fromLocal8Bit(&quot;退出&quot;)); ​ 4、通过文本编辑器将文件格式转换为UTF-8 BOM，将转换出来的文件保存，替换QT工程目录下原本的文件，再重启QT，重新构建。 ​ 我使用的是1-2和3两种方法。
